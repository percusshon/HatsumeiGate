# 発明投稿フォーム・投稿前チェック設計

## 1. 発明投稿フォーム設計

### 1.1 必須項目（投稿時必須）

| 項目 | 入力形式 | 必須 | バリデーション | 目的 |
|---|---|---:|---|---|
| 発明タイトル | 文字列 | はい | 10〜80文字 | 競合識別と要約検索 |
| 解決したい課題 | 自由記述 | はい | 100〜3000文字 | 価値検証の起点 |
| 既存品/既存サービスの不満 | 自由記述 | はい | 80〜2000文字 | 課題優位性の初期判断 |
| 発明の仕組み | 自由記述+図番号参照 | はい | 150〜4000文字 | 特徴抽出と特許性初期評価 |
| 想定ユーザー | 選択＋補足 | はい | 1件以上 | 市場適合確認 |
| 使用シーン | 自由記述 | はい | 80〜2000文字 | 具体的運用価値の確認 |
| 似ている既存製品/サービス | 自由記述 | はい | 0〜2000文字 | 先行性評価の前提 |
| 試作品の有無 | yes/no/プロトタイプのみ | はい | bool | 実現性判断 |
| 図/写真/PDF/動画/コード添付 | 複数ファイル | はい（要件付き） | 形式/容量上限あり | 技術理解と説明補強 |
| 希望する出口 | マルチセレクト | はい | 1件以上 | 商流設計 |

### 1.2 希望する出口の選択肢
- 特許化
- 実用新案
- 意匠
- 商標
- 営業秘密
- 企業売却
- ライセンス
- 共同開発

### 1.3 添付ルール（初期）
- 図面/PDF: 3000px以内/20MB以下
- 動画: 30MB以下
- コード: zip可、依存ファイルのみ
- 不要情報（従業員情報、顧客情報）を含めないこと
- 画像内に第三者商標/外部設計図が含まれる場合、権利元を明記

## 2. 投稿前チェック（submit前のゲート）

### 2.1 確認項目
1. 自分が発明者か（本人）
2. 共同発明者がいるか
3. 勤務先/委託先/学校等に権利が帰属する可能性
4. すでに公開済みか（SNS/YouTube/展示会/販売ページ/論文）
5. 他人のアイデア、資料、画像、コード、OSS混入の有無
6. 企業に見せてもよい概要が用意できるか
7. NDA前に見せる情報とNDA後のみ見せる情報を分けられるか
8. 運営へ情報提供の同意（NDA枠内情報の扱い含む）

### 2.2 ゲートロジック
- 上記1〜7が未完了だと `draft` のまま保存のみ可能
- 1〜7が完了かつ同意署名ありで `submitted` 遷移
- 1または3が「不明確」の場合、運営は`needs_more_info`へ遷移させる
- 4が明確に公開済みかつ秘密性低下が大きい場合は、開示レベルを制限

---

## 3. 投稿前同意文言（案）

### 3.1 機密取扱い同意
「投稿された情報は、運営の審査・先行技術整理・企業提案支援目的でのみ使用する。NDA前提条件に従い、企業開示はレベル管理される。

### 3.2 共同発明者同意取得同意
「共同発明者の関与がある場合、共同発明者の連絡先と権利帰属に関する情報を速やかに提供し、必要なら追加の代表権限同意を取得する。」

### 3.3 公開履歴申告同意
「SNS・ブログ・展示会・販売ページ等で公開した情報があれば、掲載内容と公開範囲を正確に申告する。」

### 3.4 誤情報免責同意
「投稿内容に第三者権利・所属権利・公開履歴の誤りがある場合、開示可否や提案対象が変更される可能性を理解する。」

### 3.5 法務的断定制限同意
「運営は特許性・商用性を断定しない。最終的な権利化・契約判断は専門家確認前提である。」

---

## 4. 投稿後の初期ステータス

- 初期ステータス: `submitted`
- 起算イベント: 同意完了 + 投稿保存 + チェック項目の必須条件を満たす
- 例外: 重要欄が不足している場合は `needs_more_info`

---

## 5. フォーム提出時の自動補助

- 初回サジェスト: 類似する既存項目の有無をヒント表示
- 競合企業リスク警告: 「同一用途の既存製品と公開情報重複が高い」場合は説明ガイド提示
- 共同発明者/所属リスクは未入力なら即時アラート表示
