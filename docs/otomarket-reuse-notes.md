# OtoMarket流用ノート（Hatsumei Gate）

## 1. 将来流用できるもの

### 認証
- メール/ソーシャル認証基盤
- SSO・二要素認証の再利用
- ユーザー権限発行の初期骨格

### Supabase client / middleware
- DB接続、セッション管理、ロールガードの雛形
- RLSの基本思想（tenant・roleベース）

### role/permission
- 管理者/通常ユーザー/法務ロールの共通概念
- APIルート保護ルール

### admin layout
- 監査メニュー・案件一覧UIの骨組み

### form components
- バリデーション付きフォームUI基盤
- ファイル添付コンポーネント

### file upload
- 画像/動画/PDFアップロードのストレージ接続

### legal page layout
- 法務文面・同意エリアの構造化UI

### support/contact
- お問い合わせ/相談窓口UI

### audit log
- 操作・閲覧・変更履歴の保存仕様（event log）

### preflight/lint/test
- CIテンプレート、品質ゲートの再利用

### Vercel設定
- デプロイ環境、環境変数管理、preview flow

---

## 2. 流用してはならないもの

- 商品販売フロー（決済・在庫・注文）
- 即時公開マーケットのUX
- ダウンロード販売
- 低手数料訴求文言
- OtoMarket固定の利用規約（対象・同意内容が一致しない）
- 販売者/購入者のままの権利関係モデル

---

## 3. Hatsumei Gateで新規実装が必要なもの

- 発明投稿フォーム（ステップ型、必須項目多数）
- 秘密保持同意管理（NDA前後2層）
- 公開済み確認フロー（SNS/論文/展示会/ページ）
- 共同発明者/勤務先権利確認フロー
- 先行技術調査メモ（運営用）
- 知財方針整理（再設計含む）
- 弁理士相談準備（相談パッケージ）
- 企業開示リクエストと開示レベル制御
- 交渉管理（譲渡/ライセンス/共同開発）
- 失敗ケースの明示（不採用/辞退/取り下げ）

---

## 4. 抽出可能性の設計メモ

- 機能を「共通/共通化困難/新規」にタグ付けし、後日モジュール化を実施
- 流用しやすいのは UI コンポーネント、監査、認証、upload、ルーティング
- 流用しづらいのは法律リスクに絡む文言・業務判断ロジック
