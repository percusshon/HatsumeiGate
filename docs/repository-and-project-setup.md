## 5. AGENTS.md（開発ルール案）

- 既存仕様を壊さないことを最優先
- ドキュメント更新時は影響節囲を明記
- 仕様変更は1回のPRあたり1テーマ
- まず定義→例外処理→監査項目の順で更新
- 重大な法務表現はレビュー必須
- 機密情報（発明内容）をサンプル化せず、実データの掲載を避ける
