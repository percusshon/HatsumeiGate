# 発明案件ステータス設計

## 1. 共通監査ログ仕様（全ステータス共通）

監査ログは必須:
- `entity_id`（発明ID）
- `invention_status`
- `status_from`
- `status_to`
- `actor_role`（inventor / operator / admin / system）
- `actor_id`
- `changed_at`
- `reason_code`
- `notes`
- `attachments`
- `ip_block_level_after`（開示レベル変更有無）
- `consents`（required consent IDs）

法令上の観点では、NDA情報や会社名開示などの特定イベントは、別監査テーブルで二重記録する。

---

## 2. ステータス定義

### 2.1 draft
- **意味**: 投稿下書き。投稿前の編集可状態
- **誰が変更できるか**: 発明者、運営（緊急修正のみ）
- **遷移可能**: `submitted`, `withdrawn`
- **遷移入力**: 投稿前チェック回答、NDA方針の暫定同意、タイトル/要約
- **監査ログ**: 保存内容スナップショット、変更者、未完了必須項目一覧
- **発明者表示**: 「下書き状態です。投稿前チェックが完了すると審査へ進みます。」
- **管理者メモ**: 未入力欄、共有資料不足、公開履歴の有無
- **通知対象**: なし（任意で運営ドラフト通知）

### 2.2 submitted
- **意味**: 投稿確定。一次審査待ち
- **誰が変更できるか**: 発明者（自己取り下げ含む）、運営
- **遷移可能**: `screening`, `needs_more_info`, `rejected`, `withdrawn`
- **遷移入力**: 投稿フォーム必須項目のバリデーション結果、同意文言同意履歴
- **監査ログ**: 提出者ID、提出時点IP、添付ファイルハッシュ、公開可能情報判定
- **発明者表示**: 「受理しました。審査チームが評価を開始します。」
- **管理者内部メモ**: 初見リスク、関連領域タグ、優先度初期値
- **通知対象**: 運営通知（新規受付）、発明者通知（受理）

### 2.3 screening
- **意味**: 一次診断中
- **誰が変更できるか**: 運営、割当レビュアー
- **遷移可能**: `prior_art_research`, `needs_more_info`, `rejected`, `withdrawn`
- **遷移入力**: screening スコア（7軸）とコメント、追加質問一覧
- **監査ログ**: 0〜5評価値、審査者ID、判定根拠、レビュー時間
- **発明者表示**: 「審査中です。要点は確認でき次第、補足依頼を行う場合があります。」
- **管理者内部メモ**: 競合類似性、独自性見立て、リスクハイ項目
- **通知対象**: 運営、発明者（補足要請時のみ）

### 2.4 needs_more_info
- **意味**: 追加情報が必要
- **誰が変更できるか**: 運営、発明者
- **遷移可能**: `screening`, `prior_art_research`, `withdrawn`, `rejected`
- **遷移入力**: 追加質問への回答、補足資料、共同発明者情報更新
- **監査ログ**: 質問履歴、回答期限、更新回数、未回答理由
- **発明者表示**: 「追加情報が必要です。指定の項目を更新すると進めます。」
- **管理者内部メモ**: 情報ギャップ、期限、再申告内容
- **通知対象**: 発明者、運営

### 2.5 prior_art_research
- **意味**: 先行技術調査メモ作成中
- **誰が変更できるか**: 運営（レビュー担当）
- **遷移可能**: `ip_strategy_review`, `needs_more_info`, `rejected`
- **遷移入力**: 検索クエリ、類似特許/公開資料、差異整理
- **監査ログ**: 検索日付、参照資料リンク、差分要約、信頼度
- **発明者表示**: 「技術的な既存例を確認中です。」
- **管理者内部メモ**: 重要先行例、回避設計候補、公開時点情報
- **通知対象**: 運営、必要時発明者

### 2.6 ip_strategy_review
- **意味**: 知財方針の暫定整理中
- **誰が変更できるか**: 運営、必要に応じ外部アドバイザーの入力
- **遷移可能**: `prototype_review`, `attorney_review_ready`, `company_disclosure_ready`, `needs_more_info`, `rejected`
- **遷移入力**: 論点ごとの判定（特許候補/実用新案/意匠/営業秘密等）
- **監査ログ**: 候補ルート、除外候補、再設計推奨有無
- **発明者表示**: 「知財方針を整理中です。公開・出願方針の方向性を提示します。」
- **管理者内部メモ**: 断定表現NGチェック、外部専門家要相談有無
- **通知対象**: 運営、発明者

### 2.7 prototype_review
- **意味**: 試作・検証性の確認中
- **誰が変更できるか**: 運営、発明者
- **遷移可能**: `attorney_review_ready`, `ip_strategy_review`, `needs_more_info`, `rejected`
- **遷移入力**: 試作品有無、実証条件、再現性メモ
- **監査ログ**: 試作状態、必要部品、評価記録、リスクコメント
- **発明者表示**: 「実装可能性の確認を進めています。」
- **管理者内部メモ**: コスト見積、再現ハードル、改良提案
- **通知対象**: 運営、発明者

### 2.8 attorney_review_ready
- **意味**: 弁理士相談前提の準備完了
- **誰が変更できるか**: 運営、発明者
- **遷移可能**: `company_disclosure_ready`, `prototype_review`, `needs_more_info`, `rejected`
- **遷移入力**: 相談用サマリ、先行技術差分、図解、公開制御マトリクス
- **監査ログ**: 相談要旨、提出資料、相談日程候補
- **発明者表示**: 「弁理士相談準備が整いました。」
- **管理者内部メモ**: どの論点を弁理士に優先して説明するか
- **通知対象**: 発明者（準備完了通知）、運営

### 2.9 company_disclosure_ready
- **意味**: 企業開示前準備完了（NDA条件待ち/対象選定）
- **誰が変更できるか**: 運営
- **遷移可能**: `company_reviewing`, `needs_more_info`, `rejected`
- **遷移入力**: 開示レベル定義、同意条件、候補企業リスト
- **監査ログ**: 開示対象会社、NDA待ち状況、企業区分（競合管理）
- **発明者表示**: 「企業向け開示準備が完了し、希望条件に合う企業へ案内可能です。」
- **管理者内部メモ**: 競合回避条件、公開可否フラグ
- **通知対象**: 運営

### 2.10 company_reviewing
- **意味**: 企業閲覧・交渉前段階
- **誰が変更できるか**: 運営、企業ユーザーの閲覧イベント、場合により発明者
- **遷移可能**: `negotiating`, `needs_more_info`, `rejected`
- **遷移入力**: 企業ごとの開示履歴、面談要否、競合制約条件
- **監査ログ**: 企業閲覧ログ、閲覧時間、ログイン単位での同意状態、閲覧レベル
- **発明者表示**: 「1社以上が検討中です。進行状況はダッシュボードで確認できます。」
- **管理者内部メモ**: 検討企業、意思決定者、リスクメモ
- **通知対象**: 発明者、運営、該当企業

### 2.11 negotiating
- **意味**: 条件交渉中（NDA完了済み）
- **誰が変更できるか**: 運営、企業管理者、発明者
- **遷移可能**: `licensed`, `assigned`, `joint_development`, `rejected`
- **遷移入力**: 提案条件（価格/期間/独占性/地域/ロイヤリティ）、内部承認記録
- **監査ログ**: 条件差分、再提案回数、法務確認フラグ
- **発明者表示**: 「現在、条件交渉段階です。条件更新で随時反映されます。」
- **管理者内部メモ**: 受け入れ可能下限、譲歩可能条件
- **通知対象**: 発明者、運営、企業担当（各提案時）

### 2.12 licensed
- **意味**: 非独占/独占ライセンス契約成立
- **誰が変更できるか**: 運営、発明者
- **遷移可能**: `archived`
- **遷移入力**: 契約ファイルID、成立日、ロイヤリティ/ロット条件、監督期限
- **監査ログ**: 契約種別、対価確定条件、法務承認番号
- **発明者表示**: 「ライセンス契約を締結しました。詳細管理画面で条件確認が可能です。」
- **管理者内部メモ**: 継続監視項目、改訂条項
- **通知対象**: 発明者、運営、会社関係者

### 2.13 assigned
- **意味**: 全権移転（譲渡）成立
- **誰が変更できるか**: 運営、発明者
- **遷移可能**: `archived`
- **遷移入力**: 買取代金、権利移転確認、再発明禁止条項有無
- **監査ログ**: 移転対象権利、対価条件、移転日、反映日
- **発明者表示**: 「譲渡契約を成立しました。」
- **管理者内部メモ**: 権利移転後の情報開示制限の確認
- **通知対象**: 発明者、運営、企業

### 2.14 joint_development
- **意味**: 共同開発へ移行
- **誰が変更できるか**: 運営、発明者、企業管理者
- **遷移可能**: `archived`
- **遷移入力**: 開発責任分担、成果帰属、開示制限、試作ロードマップ
- **監査ログ**: 開発スコープ、検証責任、知財帰属条項案
- **発明者表示**: 「共同開発で進める段階に入りました。」
- **管理者内部メモ**: マイルストーン監視、退出条件
- **通知対象**: 発明者、運営、企業

### 2.15 rejected
- **意味**: 審査不採用（継続見送り）
- **誰が変更できるか**: 運営、システム（一定条件で自動）
- **遷移可能**: `archived`
- **遷移入力**: 不採用理由（カテゴリ）、代替提案（再挑戦可否）
- **監査ログ**: 判定理由、担当者、再申請可否、候補提案
- **発明者表示**: 「今回は条件に合わず見送りです。再投稿/再設計の指針は保存します。」
- **管理者内部メモ**: 主要却下要因（市場性/特許性/公開リスク/権利帰属）
- **通知対象**: 発明者、運営

### 2.16 withdrawn
- **意味**: 発明者の任意取り下げ
- **誰が変更できるか**: 発明者
- **遷移可能**: `archived`
- **遷移入力**: 取り下げ理由
- **監査ログ**: 取り下げ理由、発明者同意、再開可否
- **発明者表示**: 「公開・運用を中断しました。」
- **管理者内部メモ**: 再開条件、過去ログ保全要件
- **通知対象**: 発明者、運営

### 2.17 archived
- **意味**: 終了または凍結状態（法的保存期間管理）
- **誰が変更できるか**: 運営のみ（例外で監査管理者）
- **遷移可能**: 原則なし（再起票は別ID）
- **遷移入力**: アーカイブ理由、保存期間、復元禁止条件
- **監査ログ**: 終了理由、最終閲覧日、法的保持要件
- **発明者表示**: 「案件はクローズしました。再度利用する場合は新規投稿をご案内します。」
- **管理者内部メモ**: 保持期限、削除制御、監査要件
- **通知対象**: 発明者、運営

---

## 3. ステータス遷移ガイド（要約）

- `draft` -> `submitted`（投稿確定）
- `submitted` -> `screening` / `needs_more_info`
- `screening` -> `prior_art_research` / `rejected`
- `prior_art_research` -> `ip_strategy_review`
- `ip_strategy_review` -> `prototype_review` / `attorney_review_ready` / `company_disclosure_ready`
- `attorney_review_ready` -> `company_disclosure_ready`
- `company_disclosure_ready` -> `company_reviewing`
- `company_reviewing` -> `negotiating`
- `negotiating` -> `licensed` / `assigned` / `joint_development` / `rejected`

---

## 4. 取引終了ルール

- `archived`, `licensed`, `assigned`, `joint_development`, `rejected`, `withdrawn` は取引終了系として、公開/開示条件を厳格凍結。
- 終了時は、既存の開示レベルを保持し、追加開示の再有効化が必要な場合は新規案件として扱う。
