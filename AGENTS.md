# AGENTS.md（hatsumei-gate）

## 1. このリポジトリでのCodex作業ルール

- 回答・作業報告は日本語で行う。
- 本リポジトリは実装フェーズ（Next.js + Supabase 導入済み）。業務ロジック docs を source of truth として最小差分で実装する。
- 既存docsの内容は必要最小限の参照に留め、勝手な大規模変更をしない。
- `git status --short` は変更後に確認する。
- コミット前に `git diff --check` を必ず実行する。
- ユーザーの明示がない限りコミットしない。
- 検証結果を作業報告に明記する。

## 2. 事業前提

- Hatsumei GateはOtoMarketとは**別ブランド／別サービス**。
- ただし将来的に共通基盤/共通部品（認証、ロール、監査など）を流用する可能性がある。
- 以下は流用しない（OtoMarket文脈の不整合回避）：
  - 販売フロー（購入・決済）
  - 即時公開マーケット
  - ダウンロード販売
  - 低手数料訴求
  - 購入者/販売者モデルをそのまま移植

## 3. Hatsumei Gateの中核業務（変更時必須確認）

- 発明投稿
- 秘密保持
- 公開済み確認
- 共同発明者確認
- 勤務先／委託先／学校への権利帰属確認
- 一次診断
- 先行技術調査メモ
- 知財方針整理
- 弁理士／弁護士接続準備
- 企業開示制御（NDA前後）
- 譲渡／ライセンス／共同開発の交渉管理

## 4. 禁止事項

- 特許取得を保証する設計・表現
- 企業売却を保証する設計・表現
- 収益化を保証する設計・表現
- 運営が弁理士業務や特許庁手続代理を行う設計
- NDA前に詳細な発明情報を企業へ開示する設計
- 発明者同意なしに詳細開示を進める設計
- 企業詳細開示を閲覧ログなしで設計すること
- 公開マーケット前提での画面・仕様設計

## 5. 実装時の原則（実装を行う段階の参照順）

- 業務ロジックdocsをソース・オブ・トゥルースとする。
- ステータス遷移: `docs/invention-status-workflow.md` に従う。
- 投稿チェック: `docs/invention-submission-checklist.md` に従う。
- 診断ロジック: `docs/invention-screening-logic.md` に従う。
- 知財分岐: `docs/ip-strategy-decision-tree.md` に従う。
- 企業開示: `docs/company-disclosure-workflow.md` に従う。
- 取引管理: `docs/deal-pipeline-workflow.md` に従う。

## 6. セキュリティ・秘密情報方針

- 発明資料は原則 private 扱い。
- public URL前提の公開設計をしない。
- 企業閲覧は、NDA / 権限 / 開示レベル / 閲覧ログを必須条件とする。
- 添付ファイル、図面、コード、動画は秘密情報として扱う。

## 7. 実装時の停止・戻しトリガー（このリポジトリ）

- 「秘密情報」「閲覧ログ」「NDAゲート」を1つでも欠ける実装は停止する。
- NDA前に level_3/4 の詳細・ファイルを企業へ開示しうる設計は停止する。
- 法的に誤認を生む表現（特許取得保証等）が混入する場合は停止する。
- OtoMarketの販売モデル（購入/ダウンロード/公開販売）UIを混入させない。
- 互換性・保存形式・スキーマ・認証・課金に関わる破壊的変更は事前に影響範囲と移行方針を明記する。
